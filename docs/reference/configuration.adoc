= Configuration reference

On this page you can find information about configuration options available in `.text2confl.yml` file as well as their alternatives in command line or env variables format.

== Configuration options

[WARNING]
====
Keep in mind the lookup order of values:

. command line argument
. environment variable
. value in `.text2confl.yml`/`.textconfl.yaml`/`text2confl.yml`/`text2confl.yaml` file (whatever is found first)

For brevity, we will use `.text2confl.yml` in examples below.
====

[cols="2,2,2,5a",options="header"]
|===
| Configuration file | CLI option | Env variable | Description

| server
| `--confluence-url`
| `CONFLUENCE_URL`
| Address of Confluence server.

NOTE: For Confluence Cloud use format `https://YOURSITE.atlassian.net/wiki`

| space
| `--space`
| `CONFLUENCE_SPACE`
| Space where documents will be uploaded. It is not possible to upload docs to multiple spaces in one run

| default-parent-id
| `--parent-id`
|
| Id of parent page where pages will be added as children if they don't have their own parent defined (either by attributes or by documents hierarchy).
Takes precedence over `default-parent`

| default-parent
| `--parent`
|
| Title of parent page where pages will be added as children if they don't have their own parent defined (either by attributes or by documents hierarchy)

| remove-orphans
| `--remove-orphans`
|
| Whether orphaned pages need to be cleaned up. By default all "managed" pages are removed.
See xref:../how-to/manage-orphaned-pages.md[dedicated page] for details

| notify-watchers
| `--notify-watchers` / `--no-notify-watchers`
|
| Send notification about updated pages. In configuration file can be set to `true` or `false`

| title-prefix
|
|
| Prefix appended to all page titles in the document tree.

E.g. prefix `(autodocs) ` turns "My page" into "(autodocs) My page"

| title-postfix
|
|
| Postfix appended to all page titles in the document tree.

E.g. postfix ` (autodocs)` turns "My page" into "My page (autodocs)"

| editor-version
| `--editor-version`
|
| Default editor version used for page rendering. Affects how pages are displayed and which features are available.
Autodetected by default. Can be overridden per page via the `editor` xref:./page-attributes.md[page property].

NOTE: Confluence Cloud only. Allowed values: `v1`, `v2`. Specify only if you need to force `v1` editor on Cloud.

| modification-check
| `--check-modification`
|
| Strategy to detect page changes. Allowed values: `hash` (default) and `content`.
See xref:#modifications-check-strategies[Modifications check strategies] below

| tenant
| `--tenant`
|
| Tenant id for uploaded pages. See xref:../how-to/configure-multi-tenancy.md[page about multi-tenancy]

| docs-location
|
|
| URL where the docs source root is located. If specified, automatically enables appending of the autogenerated note

| add-autogenerated-note
|
|
| Explicitly controls whether the autogenerated note is prepended to each page. Valid values: `true`, `false`

| autogenerated-note
|
|
| Custom text for the _note_ block prepended to each page. Must be in *Confluence Storage Format*.
See xref:#autogenerated-note[Autogenerated note] below

| auto-fix-content-tags
| `--auto-fix-content`
|
| Automatically fix misnested or unclosed HTML tags in converted content using Jsoup. Disabled by default.

When enabled, converted HTML is normalized before validation and further processing.

| code-blocks
|
|
| Section with configuration of code blocks language highlight

| code-blocks.default-language
|
|
| Default language for code block highlighting. Empty by default — Confluence will select type

| code-blocks.extra-mapping
|
|
| Custom language mapping for code blocks. Key is the language tag in the document, value is the Confluence language tag

| code-blocks.collapse-with-expand
|
|
| Wrap code blocks in an expand macro. Useful for Confluence Cloud where code blocks lack a native collapse feature.
Enabled by default for v2 editor if not specified explicitly

| markdown
|
|
| Section with Markdown-specific parameters

| client.skip-ssl
|
|
| Skip SSL certificate verification. `false` by default

| client.requests-per-seconds
|
|
| Activates request rate limiting when provided. Use if request rate is throttled by server

| client.retry-on-too-many-requests
|
|
| Retry requests when server returns 429 or 503 status codes. `true` by default
|===

=== Markdown configuration options

Markdown can be configured in `.text2confl.yml` file, in `markdown` section.

Table contains available parameters.
Dot (`.`) means that this is next level, e.g.

[source,yaml]
----
markdown:
  tables:
    column-spans: false
----

[cols="3,4,4a",options="header"]
|===
| Parameter | Default value | Description

| emoji
| `true`
| Support for link:{md_emoji}[emojis in Markdown]

| auto-links
| `true`
| Enables link:{flex_links}[auto links] — conversion of plain links into active links

| tables.column-spans
| `true`
| Controls link:{flex_tables}[COLUMN_SPANS] parameter of tables parsing

| tables.discard-extra-columns
| `true`
| Controls link:{flex_tables}[DISCARD_EXTRA_COLUMNS] parameter of tables parsing

| tables.append-missing-columns
| `true`
| Controls link:{flex_tables}[APPEND_MISSING_COLUMNS] parameter of tables parsing

| tables.header-separator-column-match
| `true`
| Controls link:{flex_tables}[HEADER_SEPARATOR_COLUMN_MATCH] parameter of tables parsing

| typography.quotes
| `false`
| Controls conversion of `"`, `'`, `<<`, `>>` into link:{flex_typography}[special html tags]

| typography.smarts
| `true`
| Controls conversion of `'`, `...`, `. . .`, `--`, `---` into link:{flex_typography}[special html tags]

| diagrams
| `true`
| Section that controls diagram generation. See link:{md_diagrams}[Markdown — diagrams]

| any-macro
| `true` if `enabled-macros` is empty, `false` otherwise
| Flag that specifies if any macro link:{simple_macros}[will be rendered] or not

| enabled-macros
| empty list
| List that specifies which macros link:{simple_macros}[will be rendered]
|===

[[asciidoc-configuration-options]]
=== AsciiDoc configuration options

AsciiDoc can be configured in `.text2confl.yml` file, in `asciidoc` section.

Table contains available parameters.
Dot (`.`) means that this is next level, e.g.

[source,yaml]
----
asciidoc:
  attributes:
    plantuml-format: png
----

[cols="2,2,5a",options="header"]
|===
| Parameter | Default value | Description

| diagrams
| `Diagrams`
| How to generate diagrams:

* `Diagrams` — activates `asciidoctor-diagrams`
* `Kroki` — relies on `asciidoctor-kroki`

| kroki.server
| not specified
| URL of the Kroki server to use, set via `kroki-server-url`

| kroki.fetch
| `true`
| Whether Kroki-generated diagrams should be attached as files to the page. Sets `kroki-fetch-diagram` attribute

| kroki.default-format
| not specified
| Sets `kroki-default-format` variable

| bundled-macros
| `true`
| Enables xref:./asciidoc/confluence-specific.adoc[asciidoctor macros for Confluence]

| attributes
| empty map
| Key-value pairs passed to Asciidoc as document attributes

| gems
| empty list
| Additional Ruby gems to _require_. Gems must be on the classpath, packaged as jar files (e.g. `asciidoctor-diagram`)

| temp-dir
| `false`
| Use a temporary directory to store all generated content

| base-dir
| `.asciidoc`
| Working directory for generated content when `temp-dir` is not enabled. Resolved relative to the document root
|===

== Additional options for `upload` command

[cols="3,2,4a",options="header"]
|===
| CLI option | Env variable | Description

| `--message`
|
| Message added as a change comment to every modified page

| `--user`
| `CONFLUENCE_USER`
| Username to use for login in Confluence

| `--password`
| `CONFLUENCE_PASSWORD`
| Password (or Cloud API token) to use for login in Confluence

| `--access-token`
| `CONFLUENCE_ACCESS_TOKEN`
| Personal access token for Confluence. Alternative to username/password

| `--skip-ssl-verification` / `--no-skip-ssl-verification`
|
| Whether to ignore invalid SSL certificate when connecting to server

| `--requests-per-second`
|
| Limits requests per second text2confl will make to Confluence APIs

| `--dry` / `--no-dry`
|
| Activate (or explicitly deactivate) dry run mode

| `--auto-fix-content`
|
| Enable automatic fixing of invalid or misnested HTML tags in converted content (opt-in). Off by default
|===
