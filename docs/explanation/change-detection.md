# Change detection strategies

text2confl avoids unnecessary uploads by checking whether a page has changed before sending it to Confluence. Two
strategies are available, each with different tradeoffs.

## `hash` strategy (default)

text2confl generates a checksum of the converted page body and stores it in Confluence page metadata during upload. On
subsequent runs, it compares the new checksum against the stored one. If they match, the page is skipped.

**Tradeoff:** The hash is stored in metadata, not the page content. If someone manually edits the page in the Confluence
editor, the metadata is unchanged - text2confl won't see the manual edit as a change, and it will not overwrite it on
the next run. This is desirable if you intentionally allow some manual edits, but means the page may drift from its
source.

## `content` strategy

text2confl compares the converted page body directly against the current page content in Confluence.

**Tradeoff:** This strategy is more sensitive to drift - any manual edit will be detected and overwritten on the next
upload. However, the comparison is currently naive and does not ignore autogenerated properties inside macro blocks,
which can cause false positives (unnecessary uploads) when macros insert dynamic content.

**Confluence Cloud limitation:** Cloud automatically normalizes page content after saving (e.g. reformatting markup,
adjusting macro properties). This means the stored content will often differ from what text2confl generated, causing
the strategy to detect a change on every run regardless of whether the source file changed. In practice, `content`
mode is only reliable on Confluence Server / Data Center with the legacy editor.

## Choosing a strategy

Use `hash` (the default) when:

- Your team policy is that Confluence pages should not be manually edited - text2confl is the only writer
- You want fast, lightweight change detection

Use `content` when:

- You are on Confluence Server / Data Center with the legacy editor
- You need text2confl to detect and overwrite manual edits automatically
- You don't have macros with autogenerated properties that change on every render

## Configuration

In `.text2confl.yml`:

```yaml
modification-check: hash     # default
# modification-check: content
```

Or per-run:

```shell
text2confl upload --docs . --check-modification content
```
