# Change detection strategies

text2confl avoids unnecessary uploads by checking whether a page has changed before sending it to Confluence. Two strategies are available, each with different tradeoffs.

## `hash` strategy (default)

text2confl generates a checksum of the converted page body and stores it in Confluence page metadata during upload. On subsequent runs, it compares the new checksum against the stored one. If they match, the page is skipped.

**Tradeoff:** The hash is stored in metadata, not the page content. If someone manually edits the page in the Confluence editor, the metadata is unchanged - text2confl won't see the manual edit as a change, and it will not overwrite it on the next run. This is desirable if you intentionally allow some manual edits, but means the page may drift from its source.

## `content` strategy

text2confl compares the converted page body directly against the current page content in Confluence.

**Tradeoff:** This strategy is more sensitive to drift - any manual edit will be detected and overwritten on the next upload. However, the comparison is currently naive and does not ignore autogenerated properties inside macro blocks, which can cause false positives (unnecessary uploads) when macros insert dynamic content.

## Choosing a strategy

Use `hash` (the default) when:
- Your team policy is that Confluence pages should not be manually edited - text2confl is the only writer
- You want fast, lightweight change detection

Use `content` when:
- You need text2confl to detect and overwrite manual edits automatically
- You don't have macros with autogenerated properties that change on every render

## Configuration

In `.text2confl.yml`:

```yaml
modification-check: hash     # default
# modification-check: content
```

Or per-run:

```shell
text2confl upload --docs . --check-modification content
```

## Autogenerated note

When `docs-location` is set in the configuration, text2confl prepends an autogenerated note to every page pointing to the source file. The default note reads:

```html
Edit <a href="__doc-root____file__">source file</a> instead of changing page in Confluence.
<span style="color: rgb(122,134,154); font-size: small;">Page was generated from source with <a
        href="https://github.com/zeldigas/text2confl">text2confl</a>.</span>
```

Two template variables are substituted:
- `__doc-root__` - the value of `docs-location` from config
- `__file__` - the path to the source file relative to the documentation root

You can customize the note text with the `autogenerated-note` configuration option. See [Configuration reference](../reference/configuration.md) for details.
